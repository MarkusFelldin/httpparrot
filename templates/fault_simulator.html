Fault Simulator

Test how your HTTP clients handle delays, slow streams, jitter, and random failures. All endpoints are rate-limited.

Delay

GET /api/delay/{seconds}
Wait N seconds (max 10), then respond with JSON. Useful for testing timeout handling.
Example
curl {{ request.url_root }}api/delay/3

                

            

        


        

            
Drip

            

                GET
                /api/drip?duration=5&numbytes=1024
                

                    Stream bytes slowly over the specified duration. Max duration 30s, max 10KB. Useful for testing streaming and progress handling.
                

                

                    
Example

                    
curl {{ request.url_root }}api/drip?duration=5&numbytes=1024

                

                

                    
                    
                    
                    
                    
                    

                        

                    

                    

                

            

        


        

            
Stream

            

                GET
                /api/stream/{n}
                

                    Stream n JSON lines (max 100), one per second. Each line contains an id, timestamp, and a random parrot status code.
                

                

                    
Example

                    
curl {{ request.url_root }}api/stream/5

                

                

                    
                    
                    
                    

                        

                    

                    

                

            

        


        

            
Jitter

            

                GET
                /api/jitter?min=100&max=2000
                

                    Random delay within a millisecond range, then respond. Max 10s (10000ms). Useful for testing variable latency tolerance.
                

                

                    
Example

                    
curl "{{ request.url_root }}api/jitter?min=100&max=2000"

                

                

                    
                    
                    
                    
                    
                    

                        

                    

                    

                

            

        


        

            
Unstable

            

                GET
                /api/unstable?failure_rate=0.5
                

                    Randomly return 200 or 500 based on the failure rate (0.0 to 1.0). Useful for testing retry logic and error handling.
                

                

                    
Example

                    
curl "{{ request.url_root }}api/unstable?failure_rate=0.3"
