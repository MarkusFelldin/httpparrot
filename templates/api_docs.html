API

Use HTTP Parrots programmatically. All endpoints support content negotiation.

{{ request.url_root }}

Interactive pages

GET /quiz - Guess the status code from the parrot image
GET /flowchart - Decision tree to pick the right status code
GET /tester - Check what status code a URL returns
GET /cheatsheet - Printable reference of all status codes

Get status code details

GET /{code}
Returns the detail page (HTML), the parrot image, or JSON depending on the Accept header.
JSON response
curl -H "Accept: application/json" {{ request.url_root }}404
Response
{ "code": "404", "description": "Not Found", "image": "/404.jpg", "meaning": "The server cannot find the requested resource...", "history": "Defined in HTTP/1.0 (RFC 1945, 1996)..." }
Accept headers
text/html HTML page
application/json JSON data
image/* Parrot image file

Get parrot image directly

GET /{code}.jpg
Returns the parrot image file directly. Useful for embedding in Markdown, Slack, READMEs, etc.
Example
curl -o parrot.jpg {{ request.url_root }}404.jpg
Markdown embed
![404 Not Found]({{ request.url_root }}404.jpg)

Random parrot

GET /random
Redirects (302) to a random status code page.
Example
curl -L {{ request.url_root }}random

Search status codes

GET /api/search?q={query}
Search status codes by code number, name, description, or keywords. Returns a JSON array of matches sorted by relevance. Useful for building Slack bots, IDE extensions, or CLI tools.
Search by keyword
curl "{{ request.url_root }}api/search?q=not+found"
Response
[ { "code": "404", "name": "Not Found", "description": "The server cannot find the requested resource...", "score": 50 }, ... ]
Search by code number
curl "{{ request.url_root }}api/search?q=40"
The score field (0-100) indicates relevance. Exact code matches score highest, followed by name matches, then description matches.

Check a URL

GET /api/check-url?url={url}
Makes a HEAD request to the given URL and returns the HTTP status code.
Example
curl "{{ request.url_root }}api/check-url?url=https://example.com"
Response
{ "code": 200, "url": "https://example.com", "headers": {"Content-Type": "text/html", "...": "..."}, "time_ms": 142 }

Return a specific status code

GET /return/{code}
Returns an actual HTTP response with the given status code and a JSON body.
Example
curl -i {{ request.url_root }}return/503
Response (HTTP 503)
{ "code": 503, "description": "Service Unavailable", "message": "This response was returned with HTTP status 503." }
Add ?delay=N to wait N seconds before responding (max 10). Rate limited.

Echo request

ANY /echo
Mirrors the incoming request back as JSON. Accepts GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS. For POST/PUT/PATCH, the request body and parsed JSON are included. Query parameters are echoed in args.
POST with JSON body
curl -X POST {{ request.url_root }}echo -H "Content-Type: application/json" -d '{"test": true}'
Response
{ "method": "POST", "url": "{{ request.url_root }}echo", "headers": {"Content-Type": "application/json", "...": "..."}, "args": {}, "body": "{\"test\": true}", "json": {"test": true} }
GET with query params
curl "{{ request.url_root }}echo?foo=bar&page=2"
Pretty-printed JSON
curl "{{ request.url_root }}echo?format=pretty"
Get a reproducible curl command
curl -X POST {{ request.url_root }}echo?format=curl -H "Content-Type: application/json" -d '{"test": true}'
Response
{"curl": "curl \\\n -X POST \\\n '{{ request.url_root }}echo' \\\n -H 'Content-Type: application/json' \\\n -d '{\"test\": true}'"}

                

            

        


        

            
Compare status codes

                /api/diff?code1={code}&code2={code}
                
Compares two HTTP status codes and returns their descriptions, categories, real-world examples, related codes, and a human-readable key difference summary.

Redirect chain

            

                GET
                /redirect/{n}
                
Returns a chain of n 302 redirects, ending at a 200 JSON response. Max 10 hops. Useful for testing redirect-following behavior.

                

                    
Example

                    
curl -L {{ request.url_root }}redirect/3

                

            

        


        

            
Mock response

            

                POST
                /api/mock-response
                
Returns an HTTP response with your specified status code, headers, and body. Useful for testing how clients handle various responses. Rate limited. See the Response Playground for an interactive UI.

                

                    
Example

                    
curl -X POST {{ request.url_root }}api/mock-response \
  -H "Content-Type: application/json" \
  -d '{"status_code": 429, "headers": {"Retry-After": "60"}, "body": "Too many requests"}'

                    
                

                

                    
Request body (JSON)

                    
{
  "status_code": 429,
  "headers": {"Retry-After": "60", "Content-Type": "text/plain"},
  "body": "Too many requests"
}

                

                
Fields: status_code (100-599, required), headers (object), body (string, max 10000 chars).

            

        


        

            
Bot integrations

            
Use the HTTP Parrots API to build Slack or Discord bots that respond with parrot images for status codes.

        


        

            
Slack bot (Python)

            

                

                    A simple Slack slash command that responds with a parrot image. When a user types /httpstatus 404, the bot replies with the 404 parrot.
                

                

                    
Flask handler for Slack slash command

                    
from flask import Flask, request, jsonify

app = Flask(__name__)
PARROT_BASE = "{{ request.url_root }}"

@app.route("/slack/httpstatus", methods=["POST"])
def slash_httpstatus():
    code = request.form.get("text", "").strip()
    if not code.isdigit():
        return jsonify({"text": "Usage: /httpstatus 404"})
    return jsonify({
        "response_type": "in_channel",
        "blocks": [{
            "type": "image",
            "image_url": f"{PARROT_BASE}{code}.jpg",
            "alt_text": f"HTTP {code} parrot",
            "title": {
                "type": "plain_text",
                "text": f"HTTP {code}"
            }
        }]
    })

                

                

                    
Slack app setup

                    

                        1. Create a Slack app at api.slack.com/apps

                        2. Add a Slash Command: /httpstatus pointing to your server's /slack/httpstatus URL

                        3. Install the app to your workspace

                        4. Type /httpstatus 404 in any channel
                    

                

            

        


        

            
Discord bot (Python)

            

                
A Discord bot using discord.py that responds to !http 404 with the matching parrot.

                

                    
discord.py bot

                    
import discord, aiohttp
from discord.ext import commands

bot = commands.Bot(command_prefix="!", intents=discord.Intents.default())
PARROT_BASE = "{{ request.url_root }}"

@bot.command()
async def http(ctx, code: str):
    """Show the HTTP parrot for a status code."""
    async with aiohttp.ClientSession() as session:
        async with session.get(
            f"{PARROT_BASE}{code}",
            headers={"Accept": "application/json"}
        ) as resp:
            if resp.status != 200:
                await ctx.send(f"Unknown status code: {code}")
                return
            data = await resp.json()

    embed = discord.Embed(
        title=f"{data['code']} {data['description']}",
        description=data.get("meaning", ""),
        color=0x00c9a7,
        url=f"{PARROT_BASE}{code}"
    )
    embed.set_image(url=f"{PARROT_BASE}{code}.jpg")
    await ctx.send(embed=embed)

bot.run("YOUR_BOT_TOKEN")

                

                

                    
Discord bot setup

                    

                        1. Create a bot at discord.com/developers/applications

                        2. Enable the Message Content intent

                        3. Install: pip install discord.py aiohttp

                        4. Run the bot and type !http 404 in any channel
