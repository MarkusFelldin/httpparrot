API

Use HTTP Parrots programmatically. All endpoints support content negotiation.

{{ request.url_root }}

Get status code details

GET /{code}
Returns the detail page (HTML), the parrot image, or JSON depending on the Accept header.
JSON response
curl -H "Accept: application/json" {{ request.url_root }}404
Response
{ "code": "404", "description": "Not Found", "image": "/404.jpg", "meaning": "The server cannot find the requested resource...", "history": "Defined in HTTP/1.0 (RFC 1945, 1996)..." }
Accept headers
text/html HTML page
application/json JSON data
image/* Parrot image file

Get parrot image directly

GET /{code}.jpg
Returns the parrot image file directly. Useful for embedding in Markdown, Slack, READMEs, etc.
Example
curl -o parrot.jpg {{ request.url_root }}404.jpg
Markdown embed
![404 Not Found]({{ request.url_root }}404.jpg)

Random parrot

GET /random
Redirects (302) to a random status code page.
Example
curl -L {{ request.url_root }}random

Check a URL

GET /api/check-url?url={url}
Makes a HEAD request to the given URL and returns the HTTP status code. Useful for checking if a URL is up.
Example
curl "{{ request.url_root }}api/check-url?url=https://example.com"
Response
{ "code": 200, "url": "https://example.com" }
